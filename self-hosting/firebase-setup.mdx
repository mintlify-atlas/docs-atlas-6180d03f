CCReward uses Firebase for two purposes:

- **Authentication** — Google Sign-in and Anonymous Authentication
- **Firestore** — user data storage

Follow the steps below to create a Firebase project and configure it for your self-hosted instance.

<Warning>
  Never commit your Firebase configuration values (API keys, project IDs, etc.) to version control. Store them exclusively in `.env.local`, which is listed in `.gitignore`.
</Warning>

## Create and configure a Firebase project

<Steps>
  <Step title="Create a new Firebase project">
    1. Open the [Firebase Console](https://console.firebase.google.com).
    2. Click **Add project**.
    3. Enter a project name (for example, `ccreward-selfhosted`) and follow the setup wizard.
    4. You can disable Google Analytics for a minimal setup, or enable it if you want usage tracking.
    5. Click **Create project**.
  </Step>

  <Step title="Add a web app to your project">
    1. On the project overview page, click the web icon (`</>`).
    2. Enter an app nickname (for example, `ccreward-web`).
    3. Do not enable Firebase Hosting — CCReward is deployed to Cloudflare Workers.
    4. Click **Register app**.
    5. Firebase will display your app's configuration object. Copy all the values — you will need them in the next step.

    The configuration object looks like this:

    ```javascript
    const firebaseConfig = {
      apiKey: "AIza...",
      authDomain: "your-project.firebaseapp.com",
      projectId: "your-project-id",
      storageBucket: "your-project.firebasestorage.app",
      messagingSenderId: "123456789",
      appId: "1:123456789:web:abc123",
      measurementId: "G-XXXXXXXXXX"
    };
    ```

    Click **Continue to console** when done.
  </Step>

  <Step title="Enable Google Sign-in">
    1. In the Firebase Console, navigate to **Authentication** > **Sign-in method**.
    2. Click **Google** under the "Native providers" section.
    3. Toggle **Enable**.
    4. Set a **Project support email** (required by Google OAuth).
    5. Click **Save**.

    <Note>
      You may be prompted to configure an OAuth consent screen in Google Cloud Console. Follow the prompts to set it up — select **External** for personal projects.
    </Note>
  </Step>

  <Step title="Enable Anonymous Authentication">
    1. Still on the **Authentication** > **Sign-in method** page, click **Anonymous**.
    2. Toggle **Enable**.
    3. Click **Save**.

    Anonymous authentication allows users to calculate rewards without signing in, which is a core feature of the app.
  </Step>

  <Step title="Enable Firestore Database">
    1. In the Firebase Console, navigate to **Firestore Database**.
    2. Click **Create database**.
    3. Select a **location** closest to your users (for example, `asia-south1` for India, `asia-southeast1` for Singapore).
    4. Select **Start in production mode**.
    5. Click **Enable**.

    <Warning>
      Starting in production mode means all reads and writes are denied by default. You must configure security rules (see below) before the app will function correctly.
    </Warning>
  </Step>

  <Step title="Copy Firebase config to .env.local">
    Using the configuration values copied in step 2, populate your `.env.local` file:

    ```bash
    NEXT_PUBLIC_FIREBASE_API_KEY=AIza...
    NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
    NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
    NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project.firebasestorage.app
    NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
    NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abc123
    NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID=G-XXXXXXXXXX
    ```

    See [Environment variables](/self-hosting/environment-variables) for the full list of required variables.
  </Step>
</Steps>

## Firestore security rules

After creating the database in production mode, configure security rules to allow authenticated users to read and write only their own data.

Navigate to **Firestore Database** > **Rules** in the Firebase Console and replace the default rules with the following:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only read and write their own document
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }

    // Deny all other access by default
    match /{document=**} {
      allow read, write: if false;
    }
  }
}
```

Click **Publish** to apply the rules.

<Tip>
  Use the [Firebase Emulator Suite](https://firebase.google.com/docs/emulator-suite) during local development to test security rules without affecting production data.
</Tip>

## Verifying the setup

After starting the development server with `npm run dev`:

1. Open the app in your browser.
2. Attempt to sign in with Google — a popup or redirect flow should appear.
3. Check the browser console for any Firebase configuration warnings. If you see `Missing required environment variables`, verify that all `NEXT_PUBLIC_FIREBASE_*` values are correctly set in `.env.local` and restart the dev server.

See [Troubleshooting](/self-hosting/troubleshooting) if you encounter authentication or database access errors.
